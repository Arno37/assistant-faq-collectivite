Rapport de Choix Stratégique

Assistant FAQ Intelligent pour Collectivité Territoriale

1. Introduction

L'objectif de ce projet est de doter la collectivité d'un assistant capable de répondre aux questions des administrés avec une fiabilité totale, tout en restant capable de comprendre le langage naturel. Après évaluation, la Stratégie B (RAG - Retrieval Augmented Generation) a été sélectionnée.

2. La Stratégie Choisie : Le RAG

Le RAG est une architecture qui connecte un Modèle de Langage (LLM) à une base de connaissances locale (nos fichiers de mairie).

Pourquoi cette stratégie est supérieure ?
  • Fiabilité (Anti-Hallucination) : L'IA est contrainte de répondre uniquement à partir des documents fournis.
  • Intelligence Sémantique : Compréhension du sens global grâce aux Embeddings.
  • Maintenance Simplifiée : Mise à jour instantanée par modification de fichiers simples.

3. Justification Technique

Le projet s'appuie sur des composants de pointe :

  • Modèle d'Embeddings : all-MiniLM-L6-v2 pour la recherche sémantique.
  • Modèle de Génération : Qwen2.5-7B-Instruct pour la formulation des réponses.
  • Contrôle : Prompt système strict assurant l'étanchéité des sources.

4. Preuves de Fiabilité (Tests Automatisés)

A. Validation Unitaire (Nettoyage) PASSED

Fichier : tests/unit/test_mono_fonction.py

Vérifie que les erreurs de frappe (majuscules, accents) sont corrigées en amont pour garantir la qualité de la recherche.

B. Validation d'Intégration (Pipeline) PASSED

Fichier : tests/integration/test_chaine_complete.py

Prouve que la lecture des fichiers réels sur le serveur vers le cerveau de l'IA est 100% opérationnelle.

5. Conclusion

La Stratégie B (RAG) est la solution la plus sûre et la plus évolutive. Elle allie la rigueur administrative à la modernité numérique.
